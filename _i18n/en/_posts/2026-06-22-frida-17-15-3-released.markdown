Quick bug-fix release, shipped later the same day because software is hard and
apparently enjoys reminding us.

- darwin: Populate the module registry before activating the interceptor. The
  first gum_interceptor_obtain() activates the unwind broker, whose backend
  resolves libdyld through the module registry. Doing this before taking the
  initial snapshot could re-enter an empty registry and crash on a NULL module
  lookup. We now snapshot the modules first, then start change tracking.
